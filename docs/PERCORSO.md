# Percorso (cosa abbiamo fatto e perché)

## 1) Terminale e comandi base
- Abbiamo usato il terminale (in VS Code o macOS) per dare comandi al computer.
- `cd` cambia cartella, `ls` lista file, `python3 ...` esegue script.

## 2) Script Python (cli/summary.py)
- Abbiamo creato uno script che legge file .txt e genera un report .summary.md.
- Serve per allenarti: input → elaborazione → output.

## 3) Git (storia delle versioni)
- `git add` = selezioni cosa includere
- `git commit -m "..."` = salvi uno “snapshot” con un messaggio
- ti permette di tornare indietro e capire cosa è cambiato

## 4) GitHub (copia online + condivisione)
- Abbiamo creato un repo su GitHub.
- Abbiamo collegato il repo locale (Mac) al repo remoto (GitHub).

## 5) Push / Pull (scambio)
- `git push` = mandi i commit su GitHub
- `git pull` = prendi aggiornamenti da GitHub sul Mac
- “upstream” = dire a Git dove deve pushare/pullare di default (origin/main)

## 6) Homebrew + gh (perché li abbiamo installati)
- Homebrew (`brew`) = installatore di programmi da terminale su macOS
- GitHub CLI (`gh`) = comandi GitHub da terminale (login, create repo, ecc.)
- Ci hanno sbloccato login e creazione repo in modo più semplice.
